Résumé des commandes Docker

Lancer un container

Le container se lance, éxecute la commande renseigné dans le Dockerfile de l'image et s'arrête. Ici un contianer hello-world de docker.

Note: si la commande lance un serveur ou attend une reponse, le container ne s'arretera pas.

docker run hello-world
    Liste des arguments utiles:
  • -n ou --name hello-world : donne un nom au container
  • -p ou --publish 8080:3000 : indique les redirections de ports au sein de Docker. Une requête sur le port 8080 sera redirigée dans Docker sur le port 3000
  • -e ou --environement MYSQL_ROOT_PASSWORD=poulet : Au sein du container la variable d'enironnement MYSQL_ROOT_PASSWORD vaudra poulet
  • -d ou --detach : indique qu'on souhaite lancer le container en mode détaché. (sans la console)

Arreter un container

Ici un contianer hello-world de docker

docker stop hello-world

Relancer un container

Ici un contianer hello-world de docker

docker start hello-world

Lister les containers

Par défaut, cette commande affiche les containers en cours d'éxécution. Ajoutez l'option -a pour lister tous les containers

docker ps

Lister les images

Les images sont des bases sur lequelles on peut se reposer pour construire de nouvelles images ou juste pour lancer un container.

docker images

Supprimer une image

docker rmi 26e2df4264a

Afficher les logs d'un container

Utile pour afficher les informations retournés par une commande.

docker logs 26e2df4264a

Attacher la console d'un container

Permet d'afficher les nouvelles lignes retournés par une commande du container

docker logs 26e2df4264a
